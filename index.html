Governance Mechanisms

🔟 10th-Man Rule

Every council forces each advisor to argue against their own position. Mandatory devil's advocate prevents groupthink. Blind spots are surfaced before synthesis.

🚫 Kill Switch

One file (KILL_SWITCH) halts the entire organization instantly. Checked at the top of every script. CEO retains ultimate control.

🔐 Recipient Tiers

Tier 1 (Hard Block): Marc, Srini, Sri, Meir — drafts only, even on "send".
Tier 2: Karishma, Laksh, Anand — send only on explicit phrase.

📊 Evidence Gates

5 gates across the PDLC chain. No PRD without evidence score ≥0.7. No prototype without RICE. No blog without release note. No auto-advance.

📈 Promotion by Data

L1→L2 requires 7+ days clean draft-quality ledger + <10% substantive edits. Agents never promote themselves. CEO reviews + config commit.

🧿 Jarvis Protocol — Shared Real-Time Context NEW IN v3.0

Every specialist shares one brain file (active-context.json). Before acting, each agent reads what's happening. After completing, each agent updates what it learned. The difference between a team that meets daily vs. a team that shares a Slack channel.

Pre-Read Protocol

Every specialist reads active-context + last 5 decisions + all lessons BEFORE any tool calls. ~2s overhead for full context awareness.

Post-Write Protocol

After completing, specialists merge findings into active-context. Merge, not overwrite. Critical items, decisions, learnings all persist.

Size Guardrails

15KB max · 14-day staleness auto-archive · 60 items hard cap. Never grows unbounded. Old items age out to archive.jsonl.

Cross-Pollination

Legal-counsel finds precedent → customer-response knows to reference it. Innovation-lab spots pattern → artifact-producer uses it in next PRD.
